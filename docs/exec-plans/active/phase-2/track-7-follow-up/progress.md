## Notes

- Canonical design reference: `docs/design-docs/phase-2/track-7-context-window-management.md` (Track 7 original). This follow-up **supersedes** that doc's three-tier-in-place-transform section; the other sections (thresholds, config surface, observability, dead-letter reason) continue to apply.
- **This track assumes Track 7 (Tasks 1–13) has shipped.** Parent PR #80 was squash-merged as commit `9454390` on 2026-04-19. Production incident that surfaced every gap: task `1717c12b-aee3-4632-b29e-b3d0e269e87f`.
- **Canonical landing order: 2 → 4 → 3 → 5.** Tasks 1, 6, 7 are independent and may land anytime. Task 2 must land before Task 3 (chunking is the safety net for large `middle` regions). Task 4 must land before Task 3 (projection assumes byte-bounded journal entries). Task 5 extends the pipeline introduced in Task 3.
- **Tasks 1 and 2** both touch `summarizer.py` — serialise or use `isolation: "worktree"` per AGENTS.md §Parallel Subagent Safety. **Tasks 3 and 5** both touch `pre_model_hook.py` — the 3 → 5 serial order keeps them conflict-free. Task 4 touches `executor/graph.py` (ingestion wiring); Task 3 also touches it (pre_model_hook wiring) — land Task 4 first or use worktrees.
- **No database migrations.** Legacy Track 7 state fields (`summary_marker`, `cleared_through_turn_index`, `truncated_args_through_turn_index`) are tolerated on load and ignored going forward. `summary` stays empty on legacy checkpoints until the first post-deploy compaction fires.
- **Expected one-time KV-cache miss** on first post-deploy summarization for every in-flight task (summary is replaced, so the prefix bytes shift once). Subsequent calls re-warm normally. Acceptable.
- **Research subagent pattern** ([#84](https://github.com/shenjianan97/persistent-agent-runtime/issues/84)) is deliberately deferred to a future track.
- **Prompt caching** ([#52](https://github.com/shenjianan97/persistent-agent-runtime/issues/52)) is a separable future track; it will need to coordinate cache-breakpoint placement with the replace-style summary introduced here.
- **Tier 1 / 1.5 Console debug visibility** is deliberately out of scope — worker structured-log events (`compaction.tier3_fired`, `compaction.offload_emitted`, `compaction.option_c_replacement`) remain the operator surface.
- **Offline suite never blocks per-commit CI** — scheduled + dispatch only, per-run budget cap via cost-ledger sum.
